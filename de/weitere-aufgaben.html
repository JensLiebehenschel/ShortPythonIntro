Fizz buzz#

Eine sehr beliebte Aufgabe für Einsteiger einer Programmiersprache, ist das Fizz buzz-Problem:

  • Es sollen die Zahlen 1 bis inklusive 100 untereinander ausgegeben werden.

  • Hierbei sollen jedoch Zahlen, welche durch 3 teilbar sind, durch den String ”Fizz” ersetzt werden.

  • Zahlen, welche durch 5 teilbar sind, sollen durch den String ”Buzz” ersetzt werden.

  • Wenn eine Zahl sowohl durch 3 als auch durch 5 teilbar ist, soll die Zahl durch den String ”FizzBuzz” ersetzt werden.

Die Ausgabe soll so aussehen, wobei die rechte Spalte nur zur Veranschaulichung da ist.

Output    (Zahl)

1         1
2         2
Fizz      3
4         4
Buzz      5
Fizz      6
7         7
8         8
Fizz      9
Buzz      10
11        11
Fizz      12
13        13
14        14
FizzBuzz  15
16        16
...       ...
98        98
Fizz      99
Buzz      100

Ihre Antwort#

# Hier können Sie das FizzBuzz Problem lösen

Lösung für FizzBuzz

for number in range(1, 101):
    if number % 3 == 0 and number % 5 == 0:
        print("FizzBuzz")
    elif number % 3 == 0:
        print("Fizz")
    elif number % 5 == 0:
        print("Buzz")
    else:
        print(number)

Erklärung der obigen Lösung:

range(1, 101), da es bei 1 starten soll und bis inklusiv 100 gehen soll. (Inklusiv 100 –> Exklusiv 101 –> range(1, 101))

Zahl ist durch n teilbar –> zahl % n == 0.

Sprich: Es gibt keinen Rest bei der Division zahl / n.

Es gibt viele Lösungen für diese bekannte Programmieraufgabe. Man könnte etwa mit Strings arbeiten und bei Bedarf Fizz und Buzz konkatenieren.

Palindrome#

Ein Palindrom ist ein Wort, welches egal ob von links nach rechts oder von rechts nach links gelesen, identisch ist.

Sprich: Man kann das Wort in der Mitte des Wortes spiegeln und es verändert sich nicht.

Beispiele: radar, rentner

Wort:   R A D A R
Index:  0 1 2 3 4

Wort:   R A D A R
Index:  4 3 2 1 0

Egal ob man “RADAR” von links nach rechts oder von rechts nach links liest, verändert sich das Wort nicht.

Ihre Aufgabe ist es, eine Funktion zu schreiben, welche ein Wort als Eingabe erhält und zurückgibt, ob es sich um ein Palindrom handelt oder nicht. Falls es sich um ein Palindrom handelt, wird True zurückgegeben, ansonsten False. Um es zu vereinfachen, gehen Sie davon aus, dass nur ein Wort angegeben wird. Außerdem keine Leerzeichen und nur Kleinbuchstaben.

# Hier können Sie das Palindrom Problem lösen

Lösung für Palindrome

Die wahrscheinlich einfachste Variante. Man kehrt den String einfach um. Wenn der umgekehrte String dem eigentlichen String gleicht, handelt es sich um ein Palindrom. Um den String umzukehren, erstellt man einen leeren String und konkateniert alle Buchstaben des eigentlichen Strings, jedoch fängt man von hinten an. Deswegen reversed(range()). Man muss Konkatenieren anstatt eine Indexzuweisung zu machen, da Strings in Python immutable sind und somit man nicht einen bestimmten Buchstaben an einem beliebigen Index verändern kann.

def ist_palindrom(string):
    umgekehrter_string = ""
    for index in reversed(range(0, len(string))):
        umgekehrter_string += string[index]
    return string == umgekehrter_string

Bei einer anderen Variante, muss man den String gar nicht umkehren. Alle Vergleiche werden im selben String durchgeführt. Hier überprüft man, ob der erste Buchstabe von links dem ersten Buchstaben von rechts gleicht. Dann ob der zweite Buchstabe von links dem zweiten Buchstaben von rechts gleicht. Dies macht man bis zur Mitte. Wenn man dort ankommt und man keine unterschiedlichen Buchstaben gefunden hat, dann handelt es sich um ein Palindrom. Wenn man irgendwo einen Unterschied findet, ist es kein Palindrom und der Vergleich wird abgebrochen.

Erklärung des Codes: es_ist_ein_palindrom wird auf True gesetzt, da man am Anfang davon ausgeht, dass es sich um ein Palindrom handelt. Für die Zählervariable wird der Bereich range(0, len(string) // 2) genutzt. Mit len(string) erhält man die Länge des Wortes. Mit dem // Operator, führt man eine Division durch und rundet das Ergebnis auf den nächst kleineren Integerwert.

Man kann durch Ausprobieren überprüfen, dass die Zählervariable bei einer geraden Wortlänge bei dem linken der beiden mittleren Buchstaben endet. Bei Wörtern mit ungerader Länge, endet die Zählervariable links von dem mittleren Buchstaben, da der mittlere Buchstabe selbst nicht überprüft werden muss, da dieser genau in der Mitte ist und eine Spiegelung nichts an diesem Buchstaben ändern kann.

Um sowohl auf einen linken als auch auf einen rechten Buchstaben zuzugreifen, benutzt man die Indizes index und -(index+1). Mit dem Index -1 greift man auf das erste Element von rechts zu. Da Indizes aber bei 0 starten, muss man auf den negativen Index 1 addieren.

Die in Verbindung stehenden Paare wäre also die Indizes {0, -1}, {1, -2}, {2, -3}, {3, -4}, … Allgemein: {index, -(index+1)}. So kommt man auf die Formel.

Wenn es nun eine Stelle gibt, welche die Voraussetzung für ein Palindrom verletzt, so wird es_ist_ein_palindrom auf False gesetzt. Optional um Zeit zu sparen, kann man mit break die For-Schleife beenden, da bereits gezeigt wurde, dass es sich nicht um ein Palindrom handelt. Somit sind weitere Vergleiche nutzlos.

Schließlich gibt man zurück, ob es sich ein Palindrom handelt oder nicht.

def palindrom_von_beiden_seiten_pruefen(string):
    es_ist_ein_palindrom = True
    for index in range(0, len(string) // 2):
        if string[index] != string[-(index+1)]:
            es_ist_ein_palindrom = False
            break
    return es_ist_ein_palindrom
